# Future Expansion Strategy

Expand only when a real client need appears.

## Near Term

- Add more theme presets: Japanese restaurant, cafe, retail, services, corporate.
- Add section variants instead of new one-off components.
- Add optional multi-page routes for clients that need dedicated menu, services, or product pages.
- Add local image assets under `public/clients/client-name` when final photography is available.

## Later

- Add content collections if client content becomes larger or needs Markdown authoring.
- Add ecommerce-lite sections for featured products, categories, and checkout links.
- Add form handling through a static-friendly provider.
- Add per-client workflow matrix builds if multiple sites are deployed from the same repo.
- Add visual regression checks once there are multiple production clients.

## Avoid For Now

- A database
- A custom CMS
- Runtime server rendering
- A complex design token pipeline
- Cross-client abstraction before a second client proves the need
